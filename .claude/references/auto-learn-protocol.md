# Auto-Learn Protocol

> Automatic knowledge extraction when tasks complete. No human input needed.
> Reference for: aegis-kanban (move to DONE), mother-brain (post-task learning)

---

## Trigger

When a task's `meta.json` status changes to DONE (after all gates pass).
This protocol runs automatically -- no command invocation required.

---

## Auto-Extract Steps

### 1. Pattern Detection

Read the task's `_aegis-brain/tasks/{ID}/history.md` and extract:

- **Time per phase**: Calculate duration between status transitions
  - spec (BACKLOG->TODO), build (IN_PROGRESS duration), review (IN_REVIEW duration), qa (QA duration)
- **Gate retry count**: Count GATE_FAIL entries per gate number
- **Issue finder**: Which agent logged GATE_FAIL (Vigil? Sentinel? Havoc?)
- **Fix type**: Categorize from comments (bug fix, missing test, security, style, logic error)

### 2. Skill Match

Map the task to skills used:

- Read `meta.json` fields: `task_type`, `labels`, `points`
- Identify primary skill: map `task_type` to skill file
  - `impl` -> code patterns, `test` -> qa-pipeline, `review` -> code-review, `arch` -> architecture
- Determine what worked well (zero gate retries = clean execution)
- Determine what failed (gate retries > 0 = friction point)

### 3. Auto-Write Learning

Append to `_aegis-brain/learnings/auto-learned.md`:

```
### LEARN-{counter} | {YYYY-MM-DD} | {task_id}
- **Task**: {title} [{points}pts]
- **Duration**: {total_time} (spec:{t1}, build:{t2}, review:{t3}, qa:{t4})
- **Gate retries**: G1:{n}, G2:{n}, G3:{n}
- **Pattern**: {detected_pattern}
- **Insight**: {auto-generated insight}
- **Action**: {what to do differently next time}
```

If `auto-learned.md` does not exist, create it with header:
```markdown
# Auto-Learned Patterns
> Auto-generated by the Auto-Learn Protocol. DO NOT edit manually.
> Each entry is extracted when a task moves to DONE.
```

**Counter management**: Read the file, count existing `### LEARN-` entries, increment by 1.

### 4. Pattern Promotion

After writing a new learning, scan all entries in `auto-learned.md`:

1. Group entries by `Pattern` field (fuzzy match on similar descriptions)
2. If the same pattern appears **3+ times**:
   - Promote to `_aegis-brain/resonance/evolved-patterns.md`
   - Format the promoted entry:
     ```
     ### PAT-{counter} | Promoted: {date} | Occurrences: {N}
     - **Pattern**: {description}
     - **Evidence**: LEARN-{a}, LEARN-{b}, LEARN-{c}
     - **Confidence**: PROVEN
     - **Auto-apply**: YES -- agents should apply this pattern by default
     ```
   - Tag as PROVEN pattern
   - Mother Brain reads this on next `/aegis-start`

### 5. Anti-Pattern Detection

After writing a new learning, check for repeated failures:

1. Scan entries where `Gate retries` > 0
2. Group by failure reason (from comments/history)
3. If a gate fails **2+ times for similar reasons**:
   - Add to `_aegis-brain/resonance/anti-patterns.md`:
     ```
     ### ANTI-{counter} | Detected: {date} | Occurrences: {N}
     - **Anti-Pattern**: {description}
     - **Evidence**: LEARN-{a}, LEARN-{b}
     - **Gate affected**: G{n}
     - **Severity**: WARNING
     - **Mitigation**: {what agents should do to avoid this}
     ```
   - Tag as AVOID
   - Relevant agent gets warned on next similar task

## Integration Points

| System | How it connects |
|--------|----------------|
| aegis-kanban (move to DONE) | Triggers this protocol after successful move |
| Mother Brain (scan) | Reads evolved-patterns.md and anti-patterns.md at session start |
| Shared Intelligence | Reusable insights also written to skill-cache |
| Skill Evolution | Auto-learned data feeds into skill update decisions |
| aegis-retro | Session retro references auto-learned patterns |

---

## File Locations

| File | Purpose |
|------|---------|
| `_aegis-brain/learnings/auto-learned.md` | Raw auto-extracted learnings |
| `_aegis-brain/resonance/evolved-patterns.md` | Promoted proven patterns (3+ occurrences) |
| `_aegis-brain/resonance/anti-patterns.md` | Detected anti-patterns (2+ gate failures) |
| `_aegis-brain/skill-cache/` | Reusable cross-agent insights |
